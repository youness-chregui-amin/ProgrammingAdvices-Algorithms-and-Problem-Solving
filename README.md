### Projects

#### Project 1 – Doubly Linked List

Implemented a generic **Doubly Linked List** from scratch using C++ templates. The implementation includes fundamental list operations such as insertion, deletion, searching, updating, reversing, clearing the list, node traversal, indexed access, and size management. The project focuses on understanding pointer manipulation, bidirectional links, and dynamic memory management.

#### Project 2 – Queue Implementation Using Doubly Linked List

Built a reusable **Queue** data structure from scratch using the previously implemented Doubly Linked List as the underlying storage mechanism. The project includes standard queue operations such as push, pop, front, back, size, and empty checks, together with several additional utility functions including reverse, update, indexed access, insertion, and clearing the queue.

#### Project 3 – Stack Implementation Using Doubly Linked List

Implemented a generic **Stack** data structure from scratch using C++ templates by extending the previously developed Queue and Doubly Linked List implementations. The project follows the **LIFO (Last In, First Out)** principle and includes standard stack operations such as push, pop, top, bottom, size, and empty checks, along with additional utility functions including reverse, indexed access, updating elements, insertion, and clearing the stack. This project also demonstrates the effective use of inheritance to maximize code reusability and minimize duplication.

#### Project 4 – Dynamic Array Implementation

Implemented a generic **Dynamic Array** from scratch using C++ templates without relying on the Standard Template Library (STL). The implementation supports dynamic memory allocation, automatic resizing, indexed access, insertion at different positions, deletion, searching, reversing, clearing the array, and element updates. This project focuses on understanding contiguous memory management, dynamic allocation, and building reusable container classes that mimic the behavior of standard dynamic arrays.

#### Project 5 – Queue Implementation Using Dynamic Array

Implemented a generic **Queue** data structure from scratch using a custom **Dynamic Array** as the underlying storage mechanism instead of a linked list. The implementation supports standard queue operations including push, pop, front, back, size, and empty checks, together with additional utility functions such as indexed access, updating elements, reversing the queue, insertion at different positions, and clearing all elements. This project demonstrates how the same abstract data structure can be implemented using a different internal storage model while maintaining the same public interface.

#### Project 6 – Stack Implementation Using Dynamic Array

Implemented a generic **Stack** data structure from scratch using a custom **Dynamic Array** as the underlying storage mechanism. The implementation follows the **LIFO (Last In, First Out)** principle and provides standard stack operations including push, pop, top, bottom, size, and empty checks. Additional utility functions such as indexed access, element updates, reversing, insertion at different positions, and clearing the stack were also implemented. This project further demonstrates how the same abstract data structure can be implemented using different internal storage mechanisms while promoting inheritance, code reusability, and consistent interfaces across custom container classes.

#### Project 7 – Undo & Redo System Using Stack

Implemented a practical **Undo & Redo** system using the Stack data structure. The project maintains separate undo and redo histories to manage application state changes efficiently. It demonstrates a real-world use case of the **LIFO (Last In, First Out)** principle while reinforcing concepts such as state management, encapsulation, object-oriented design, and the practical application of stacks in software development.

#### Project 8 – Queue Line Management System

Developed a console-based **Queue Line Management System** that simulates real-world customer service queues. The project manages ticket issuance, customer waiting lines, service order, expected waiting times, and queue statistics while utilizing Queue and Stack data structures internally. It also integrates custom utility libraries, including String and Date classes, to generate ticket information and timestamps. This project demonstrates how multiple custom data structures and reusable components can be combined to build practical object-oriented applications.

Outcome

This level significantly strengthened my understanding of data structures by implementing them from scratch using multiple internal storage techniques instead of relying on the C++ Standard Template Library (STL). It improved my knowledge of templates, dynamic memory management, inheritance, encapsulation, and reusable software design while demonstrating how abstract data structures can be transformed into practical real-world applications such as Undo/Redo systems and Queue Management Systems.
